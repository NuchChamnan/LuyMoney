1. Introduction (សេចក្ដីផ្ដើម)

Project Name: LuyMoney (លុយម៉ាណេ)

Platform: Android, iOS, Web

Framework: Flutter (Dart)

Backend: Firebase (Auth, Firestore, Storage, Messaging, Analytics, Crashlytics, App Check)

State Management: GetX (MVC Pattern)

Project LuyMoney គឺជា Mobile Application សម្រាប់ Financial Education។ Application ផ្ដល់ Videos, Articles, Chat Support, Subscription Management, Admin Panel, Multi-language (English/Khmer/Chinese), Multi-theme (Black/White/Blue Vintage)។

5. Services (lib/app/services/)

Services ជា Global GetxService ដែល Register ក្នុង main() ហើយ Available ពេញ App ដោយ Get.find<ServiceName>().

📄 auth_service.dart

តួនាទីFirebase Authentication Service (Global Auth State)
ObservablesRx<UserModel?> currentUser, RxBool isLoading
GettersisLoggedIn (Firebase Auth only), isAdmin (role == admin), hasActiveSubscription, firebaseUser
MethodssignInWithEmail(), registerWithEmail(), sendPasswordResetEmail(), signOut(), updateUserProfile(), deleteUserData(), refreshUser()
TimeoutsFirebase Auth calls: 20s timeout. Firestore loads: 10s timeout (prevent infinite hang)
ដំណើការListen to authStateChanges() → _loadUserData(uid) ពី Firestore → set currentUser
សម្រាប់ធ្វើអ្វីប្រើ Get.find<AuthService>() ពីទីណាក៏បាន ដើម្បីដឹង User Info + Login status

📄 notification_service.dart

តួនាទីPush Notification Service (Local + FCM)
ដំណើការ1) Init flutter_local_notifications 2) Create Android channel 3) Request FCM permission 4) Listen onMessage 5) Handle background messages
MethodsscheduleExpiryReminders(), cancelAllReminders(), getFCMToken()
Scheduled Reminders7 days before expiry, 1 day before expiry, on expiry day
PlatformAndroid + iOS (Web not supported)

📄 storage_service.dart

តួនាទីLocal Storage Service Wrapper around GetStorage
Keystheme, lang, remember_email, biometric_enabled, notif_subscription, notif_content, notif_promo, bookmarked_videos, bookmarked_articles
Methodsread(), write(), remove(), erase(), toggleVideoBookmark(), toggleArticleBookmark(), isVideoBookmarked()

📄 analytics_service.dart

11. Firebase / Firestore Database Structure

Firestore Collections

Collection PathPurposeDocument ID
users/{uid}User profiles + roleFirebase Auth UID
chats/{uid}Chat metadata per userFirebase Auth UID
chats/{uid}/messages/{msgId}Individual chat messagesAuto-generated
videos/{videoId}Video content itemsAuto-generated
articles/{articleId}Article content itemsAuto-generated
subscriptions/{subId}User subscription recordsAuto-generated
admin_notifications/{id}Queued push notificationsAuto-generated

Security Rules (Current)

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
    match /chats/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
